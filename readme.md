# 磁贴成长纪 (Magnet Challenge)

**版本**: v3.0  
**定位**: 面向家庭的儿童习惯养成与情绪管理小程序  
**技术栈**: Taro 3 + React + Zustand + Tailwind CSS

## 1. 项目背景与目标

"磁贴成长纪"是一款基于数字化"代币制"（Token Economy）心理学原理开发的亲子互动小程序。旨在通过游戏化的方式，帮助孩子建立良好的生活习惯，培养情绪控制能力，并建立"付出-收获"的价值观念。

## 2. 核心机制

应用围绕 **双轨激励制 (Dual-Track Incentive System)** 展开，将高频小奖励与低频大梦想分离，解决孩子"延迟满足"的心理难点：
- **零花钱 - 磁贴 (Magnets)**: 孩子通过完成每日任务、控制情绪等日常行为获得。可随时用于兑换 Mini 奖励（如看电视15分钟），提供即时的正向反馈。
- **许愿星 (Star Tokens) 🌟**: 用于兑换高价值的"大梦想"玩具。不需要孩子艰难地攒几十个磁贴，而是通过每周累积获得一定数量的磁贴后，系统自动奖励一颗"许愿星"。
- **可视化激励**: 通过动画、音效和目标进度条，让孩子的每一次努力都能被看见。

## 3. 核心功能模块

### 3.1 首页 (Home) - 聚合仪表盘
- **视觉风格**: 采用蓝紫色渐变主题，营造梦幻、积极的氛围
- **状态总览**: 顶部大卡片融合了"当前磁贴总数"与"今日任务完成进度"，一目了然
- **Tab 切换**: 滑动胶囊切换器，区分"⚡ 今日挑战"和"🏆 恒心榜单"
- **响应式布局**: 适配手机竖屏，卡片自动流式排列

### 3.2 双维度任务体系 ⭐ 新功能

#### 每日任务 (Daily Tasks)
设定了5个基础生活习惯，每日自动重置：
1. **吃饭香香**: 专心吃饭
2. **洗刷刷达人**: 卫生习惯
3. **玩具回新家**: 整理收纳
4. **上学不迟到**: 守时观念
5. **准时梦游记**: 作息规律

**特色功能**：
- 支持每日多次完成（可设置每日上限）
- 即时反馈：完成任务触发磁贴飞入动画
- 全垒打奖励：当日5项任务全部完成后，自动触发全垒打特效，并额外奖励 +5 磁贴

#### 月度打卡 (Monthly Check-in) 🆕
培养长期习惯的"恒心榜单"：
- **目标设定**: 家长可设置月度目标天数（如：坚持阅读20天）
- **进度可视化**: 
  - 分段进度条（20个小圆点）
  - 数字进度显示（如 12/20）
- **双倍奖励机制**：
  - 每日打卡：+1 磁贴
  - 达成目标：额外大奖（如 +10 磁贴）
- **打卡动画**: 磁贴吸入礼物盒的精美动画
- **流光效果**: 达成目标后卡片显示金色边框流光

### 3.3 情绪与行为管理工具

#### 心情转换器 (Mood Converter)
- 引导孩子识别负面情绪（生气、难过、疲惫）
- 鼓励孩子主动调整心态，成功转换后给予 +2 磁贴奖励
- 配备庆祝动画，强化正向心理暗示

#### 磁贴时刻 (Magnet Moments)
- 用于记录每日任务之外的良好表现（如帮助做家务、分享玩具）
- 支持自定义描述和奖励数量 (1-5个)
- **编辑功能** 🆕: 可随时修改内容和磁贴数量
- **优化布局**: 操作按钮移至卡片底部，更符合直觉

### 3.4 梦想屋 (Reward Shop)
- **双货币奖励分级**:
  - **Mini奖励/小奖励**: 消耗普通"磁贴"(低成本高频次，如看电视、吃零食)
  - **大梦想**: 消耗"许愿星" ⭐ (长期目标，如买心仪大玩具)
- **兑换动画**: 礼物盒弹出 + 磁贴/许愿星吸入效果
- **家长验证**: 兑换前需通过数学题验证（10以内加法）
- **兑换逻辑**: 智能判断所选商品消耗的货币类型，余额不足时按钮提示差额，余额充足时方可兑换。

### 3.5 个人中心与家长管理 (Profile & Admin)
- **数据概览**: 展示连续打卡天数 (Streak)、全垒打次数等荣誉数据
- **许愿星进度 (Weekly Milestone)** 🆕:
  - 动态展示本周累计获得的磁贴数量。
  - 进度条满后，即可点击领取一颗"许愿星"。
- **家长验证机制 (Math Gate)**:
  - 在进行敏感操作（如进入管理后台、兑换奖励）时触发
  - **算法**: 随机生成 10 以内的加法题（和 <= 9），防止幼龄儿童误操作
- **任务管理**:
  - 区分每日任务和月度打卡
  - 可自定义任务图标、名称、奖励数值
  - 每日任务：设置每日完成上限
  - 月度任务：设置目标天数和达成奖励
- **奖励管理**: 家长可自定义奖励内容及所需磁贴数

## 4. 技术特性

### 框架与工具
- **框架**: Taro 3.6 + React 18
- **状态管理**: Zustand (轻量级、零样板代码)
- **样式**: Tailwind CSS + weapp-tailwindcss
- **动画**: 纯 CSS + Emoji 动画（高性能）
- **存储**: Taro.getStorageSync 本地持久化

### 核心技术实现

#### 状态管理 (Zustand)
```typescript
interface StoreState {
  user: User;
  tasks: Task[];
  rewards: Reward[];
  logs: Log[];
  
  // Actions
  addMagnets: (amount: number, description: string, type: Log['type']) => void;
  toggleTask: (taskId: string) => void;
  toggleMonthlyTask: (taskId: string) => void; // 月度打卡
  checkDailyReset: () => void;
  checkMonthlyReset: () => void; // 月度重置
}
```

#### 智能重置逻辑
- **每日重置**: 每天 0:00，重置每日任务的 `completed` 状态
- **月度重置**: 每月 1 号 0:00，清空月度任务的 `monthlyProgress` 和 `history`
- **数据迁移**: 自动为旧任务添加 `type: 'daily'` 字段

#### 动画系统
- **获得磁贴**: 磁贴弹出 + 光环炸裂 + 星星四散
- **兑换奖励**: 礼物盒弹出 + 磁贴吸入 + 盒盖弹开
- **全垒打**: 奖杯弹出 + 全方位庆祝效果
- **性能优化**: 纯 CSS 动画 + GPU 加速，零外部依赖

## 6. 设计规范

### 视觉风格
- **玻璃拟态**: `bg-white/80 backdrop-blur-xl rounded-3xl`
- **蓝紫渐变**: `from-brand-primary via-brand-secondary to-brand-accent`
- **童趣化文案**: 避免生硬的"操作成功"，使用"约定已生效！"

### 交互规范
- **即时反馈**: 所有点击操作必须有视觉动画 + 物理震动
- **儿童友好**: 大按钮、高对比度、清晰的图标
- **防误触**: 重要操作（兑换、扣除）需要二次确认

## 8. 开发指南

### 环境准备
```bash

# 开发模式
```

### 代码规范

- 所有 Store 更新使用 `set(state => ({ ... }))` 路径更新
- 禁止在主包存放超过 200KB 的资源
